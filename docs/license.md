# Licensing of Manush-E

## 🎯 License Strategy

- **License Used**: [GNU GPL v3](https://www.gnu.org/licenses/gpl-3.0.html)
- **Applies To**:  
  - `manush-e-frontend` repository  
  - `manush-e-backend` repository
- **Why GPLv3?**  
  GPLv3 ensures that:
  - All code modifications and derivatives must remain open and shareable under the same license.
  - Users have the freedom to run, study, share, and improve the software.
  - Downstream adopters cannot make the project proprietary, which is key to our vision of universal access.

---

## 📁 Repository Licensing Implementation

### Frontend Repo
- A `COPYING` file with full GPLv3 text is added to the root.
- `package.json` specifies `"license": "GPL-3.0"`.

### Backend Repo
- Full license text in root `COPYING`.
- Composer.json includes `"license": "GPL-3.0"`.

## Derivative Works and Reuse

We encourage reuse and adaptation in aligned projects, provided they:

- Attribute the original source (Moner Bondhu and Manush-E).
- Maintain GPLv3 licensing for all derivative works.
- Do not introduce discriminatory, exploitative, or privacy-violating features.

---

## Data and Privacy Caveat

While the **code is open**, **production data is NOT open** and must remain protected.  
Manush-E data (including mental health and child health information) is governed under strict privacy policies and **cannot** be redistributed under any open license.
